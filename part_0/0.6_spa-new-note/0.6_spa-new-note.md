// User makes a new post in Single Page Application https://studies.cs.helsinki.fi/exampleapp/spa

sequenceDiagram
    participant browser
    participant server
    
    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    activate server
    server-->>browser: [    {content: "yiihaaa", date: "2026-04-07T07:03:23.897Z"}  ]
    deactivate server

    Note: On browser,  Request headers informs the server that the message content type is JSON and the status code is 201 Created.

    In the beginning Browser only retrieves the list of notes from server. When a new note has been added it executes the Javascript function for adding a new DOM element and makes POST request. The difference is that in SPA, creating a new message does not fire up a new GET request on server for displaying the updated messages.
